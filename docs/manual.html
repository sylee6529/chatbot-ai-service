1. 서비스 개요

이 API는 고객사의 앱, 웹, 내부 업무 도구에서 AI 챗봇 답변을 호출할 수 있게 하는 백엔드 인터페이스입니다.

  • 사용자는 질문을 전송하고 AI 답변을 받을 수 있습니다.
  • 생성된 질문과 답변은 대화 이력으로 저장되며, 이어지는 대화는 스레드 단위로 관리됩니다.
  • 관리자는 전체 대화와 피드백을 확인하고, 최근 24시간의 사용자 활동 및 대화 보고서를 조회할 수 있습니다.
구분설명
인증회원가입과 로그인으로 사용자를 식별하고, 로그인 후 발급된 JWT 토큰으로 API 요청을 인증합니다.
대화질문을 입력하면 AI 답변을 생성하고, 질문과 답변을 하나의 채팅으로 저장합니다.
대화 이력30분 이내의 후속 질문은 같은 스레드에 저장되어 이전 대화 흐름을 유지합니다.
피드백사용자는 대화에 대한 피드백을 생성할 수 있으며, 관리자는 피드백 처리 상태를 변경할 수 있습니다.
관리자 기능관리자는 활동 집계와 CSV 보고서를 통해 서비스 사용 현황을 확인합니다.

2. 주요 용어

용어의미
사용자API를 사용하는 계정입니다. 일반 회원은 MEMBER, 관리자는 ADMIN 권한을 가집니다.
JWT 토큰로그인 후 발급되는 인증 문자열입니다. 보호 API 호출 시 Authorization 헤더에 넣습니다.
채팅질문 1개와 AI 답변 1개가 저장된 단위입니다.
스레드이어지는 채팅을 묶은 대화 단위입니다. 30분 이내의 후속 질문은 같은 스레드에 저장됩니다.
피드백AI 답변이 도움이 되었는지 긍정 또는 부정으로 기록하는 값입니다.

대화 예시

아래 내용은 질문과 답변이 어떻게 구성되는지 보여주기 위한 예시입니다.

업무 주제예시 내용질문 예시
VIP 고객 온보딩 정책 신규 고객사는 영업 담당자 승인 후 3영업일 이내 샌드박스 키를 받습니다. “VIP 고객 온보딩은 며칠 안에 완료되나요?”
API 이용 가이드 모든 보호 API는 JWT Bearer 토큰을 사용하며, 토큰 만료 시 재로그인이 필요합니다. “API 호출할 때 인증은 어떻게 하나요?”
장애 대응 안내 AI provider 응답 실패는 502, 시간 초과는 504로 구분합니다. “AI 응답이 늦으면 어떤 오류가 오나요?”
대화 예시
VIP 고객 온보딩은 며칠 안에 완료되나요?
예시 정책 기준으로, 영업 담당자 승인 후 3영업일 이내 샌드박스 키가 발급됩니다. 이후 고객사는 API 인증 토큰을 발급받아 챗봇 API를 호출할 수 있습니다.
그 다음 API 인증 방식도 알려줘.
회원가입과 로그인 후 받은 JWT를 Authorization: Bearer 형식으로 전달합니다. 같은 스레드 안에서는 이전 질문 맥락을 이어서 답변합니다.

4. 계정 생성과 로그인

회원가입과 로그인을 제외한 모든 요청은 JWT 토큰이 필요합니다. 로그인에 성공하면 응답의 accessToken을 이후 요청 헤더에 넣습니다.

4.1 회원가입

POST /api/v1/auth/signup

권한: 인증 없이 호출할 수 있습니다.

curl -X POST http://localhost:8080/api/v1/auth/signup \
  -H 'Content-Type: application/json' \
  -d '{"email":"alice@example.com","password":"Pw123456!","name":"Alice"}'
{
  "id": 1,
  "email": "alice@example.com",
  "name": "Alice",
  "role": "MEMBER",
  "createdAt": "2026-06-27T05:00:00Z"
}

4.2 로그인

POST /api/v1/auth/login

권한: 인증 없이 호출할 수 있습니다.

TOKEN=$(curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"alice@example.com","password":"Pw123456!"}' | jq -r .accessToken)
{
  "accessToken": "eyJhbGciOiJIUzI1NiI...",
  "tokenType": "Bearer",
  "user": {
    "id": 1,
    "email": "alice@example.com",
    "name": "Alice",
    "role": "MEMBER"
  }
}

이후 보호 API는 아래 헤더를 사용합니다.

Authorization: Bearer <accessToken>

5. 질문과 답변 생성

POST /api/v1/chats

권한: 로그인한 사용자가 호출할 수 있습니다. 일반 사용자는 자신의 계정으로 대화를 생성하고, 관리자는 관리자 계정으로 동일하게 호출할 수 있습니다.

  • 질문을 보내면 AI 답변, 채팅 ID, 스레드 ID가 반환됩니다.
  • model을 생략하면 서버 기본 모델을 사용합니다.
  • AI_API_KEY가 없으면 외부 AI를 호출하지 않고 키 미설정을 명시한 fallback 답변을 반환합니다.
curl -X POST http://localhost:8080/api/v1/chats \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{
    "question": "VIP 고객 온보딩은 며칠 안에 완료되나요?",
    "model": "gpt-4o-mini",
    "isStreaming": false
  }'
{
  "chatId": 10,
  "threadId": 3,
  "question": "VIP 고객 온보딩은 며칠 안에 완료되나요?",
  "answer": "예시 정책 기준으로, 영업 담당자 승인 후 3영업일 이내 샌드박스 키가 발급됩니다.",
  "model": "gpt-4o-mini",
  "createdAt": "2026-06-27T05:01:00Z"
}

스트리밍 응답

curl -N -X POST http://localhost:8080/api/v1/chats \

6. 대화 이력 관리

  • 스레드는 이어지는 대화를 묶는 단위입니다.
  • 같은 사용자의 마지막 질문으로부터 30분 이내에 다시 질문하면 기존 스레드를 유지하고, 30분이 지나면 새 스레드를 만듭니다.
  • 응답의 threadId가 이전 질문과 같으면 같은 대화 맥락이 유지된 것입니다.
  • 이 스레드의 이전 질문과 답변은 다음 AI 요청의 문맥으로 함께 전달됩니다.

스레드별 대화 목록

스레드 삭제

DELETE /api/v1/threads/{threadId}

권한: 일반 사용자는 본인이 생성한 스레드만 삭제할 수 있습니다. 관리자는 모든 스레드를 삭제할 수 있습니다.

삭제된 스레드는 일반 조회, AI 문맥, 피드백 목록, CSV 보고서에서 제외됩니다.

7. 답변 피드백 관리

  • 특정 AI 답변에 대한 사용자 피드백을 기록하는 API입니다.
  • 피드백 값은 긍정 또는 부정으로 저장됩니다.
  • 한 사용자는 하나의 대화에 한 번만 피드백을 생성할 수 있습니다.

POST /api/v1/chats/{chatId}/feedbacks

권한: 일반 사용자는 본인이 생성한 대화에만 피드백을 생성할 수 있습니다. 관리자는 모든 대화에 피드백을 생성할 수 있습니다.

curl -X POST http://localhost:8080/api/v1/chats/10/feedbacks \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"isPositive":true}'
{
  "id": 5,
  "chatId": 10,
  "userId": 1,
  "isPositive": true,
  "status": "pending",
  "createdAt": "2026-06-27T05:12:00Z"
}

관리자 처리 상태 변경

PATCH /api/v1/feedbacks/{id}/status

권한: 관리자만 피드백 상태를 변경할 수 있습니다.

curl -X PATCH http://localhost:8080/api/v1/feedbacks/5/status \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"status":"resolved"}'

9. 오류 처리

코드상황고객사 조치
400질문이 비어 있거나 요청 형식이 잘못됨JSON 필드명과 필수값을 확인합니다.
401토큰이 없거나 만료됨다시 로그인하여 새 토큰을 사용합니다.
403본인 리소스가 아니거나 관리자 전용 API 호출계정 권한과 대상 ID를 확인합니다.
404채팅, 스레드, 피드백을 찾을 수 없음ID가 올바른지, 삭제된 항목이 아닌지 확인합니다.
409이메일 또는 피드백 중복기존 가입 여부 또는 이미 남긴 피드백인지 확인합니다.
502AI provider 응답 오류서버 운영자에게 provider 상태 확인을 요청합니다.
504AI provider 응답 시간 초과잠시 후 재시도합니다.
